Key Accessibility Concepts

Accessibility Fundamentals
Designing for all abilities using tools like screen readers, keyboard navigation, captions, and more. Includes the principles: Perceivable, Operable, Understandable, and Robust.
Semantic HTML
Meaningful elements like <header>, <nav>, <main>, <footer>, and proper heading levels help assistive tech understand the structure.
ARIA
Adds extra information to custom UI controls when semantic HTML is not enough. Use sparingly—HTML elements should always come first.
Accessible Forms
Labels must be connected to inputs. Use fieldsets, legends, clear help text, and readable error messages. Everything should work by keyboard.
Color and Contrast
Text must have enough contrast with the background. WCAG recommends 4.5:1 for normal text and 3:1 for large text. Don’t rely on color alone to show meaning.
Keyboard Navigation
Users should be able to tab through links, buttons, and form fields in a logical order. Custom controls must also work without a mouse.
Screen Readers
Screen readers use structure (headings, landmarks, labels, ARIA) to read content. Testing with one helps catch issues visual users won’t see.

Quick Accessibility Checklist

  • Is there one clear <h1> on the page?
  • Are headings and sections structured clearly?
  • Does keyboard tabbing reach everything?
  • Do all meaningful images have alt text?
  • Are all form inputs labeled?
  • Does all text pass color contrast standards?
  • Do custom widgets communicate correctly with screen readers?
